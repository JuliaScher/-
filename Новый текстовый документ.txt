TCP,UDP,ICMP(пакеты передаются без заголовков для каждого сканируемого хоста,
если получено сообщение о недоступности-не поддерживается, иначе-поддерживается):
nmap -sO 192.168.1.1
TCP Connect() все tcp порты(пытается установить соединение, успешно-открыт):
nmap -sT 192.168.1.1
tcpdump 0i enp0s8 tcp 192.168.1.1
Все UDP порты:
nmap -sU 192.168.1.1
Объединение:
nmap -p U:53,79,113,T:21-25,80,443,8080 192.168.1.1 
быстрое сканирование:
nmap -F 192.168.1.1
определение ОС:
nmap -O 192.168.1.1
версии сервисов:
nmap -sV 192.168.1.1
ACK(отправляются ack пакеты, если ответ rst-не фильтруется,если нет ответа или icpm-фильтруемый):
nmap -sA 192.168.1.1
TCP Windows(по значениям поля initial windows tcp пришедшего в ответ-определить open port):
nmap - sW 192.168.1.1
FIN,XmasTree,NULL(определение семейство ОС, windows никогда не ответит rst):
nmap -sF/-sX/-sN 192.168.1.1
TCP-SYN(посылает syn пакет для установка нового соединения, если ответ syn или ack-open, rst-close):
nmap -sS 192.168.1.1
RPC(определение программы и версии, обслуживающей порт и заключается в затоплении):
nmap -sR 192.168.1.1
никогда не произведет отбратное преобразование dns имен для каждого обнаруженного активного ip адреса:
nmap -n 192.168.1.1
подмена mac-адреса или случайным:
nmap --spoof-mac 0/00:11:22:33:44:55 192.168.1.1

сканировать множество ip адерсов:
nmap 192.168.1.1 192.168.1.2 192.168.1.3
nmap 192.168.1.1,2,3
сканировать подсеть:
nmap 192.168.1.0/24
nmap 192.168.1.*
сканировать диапазон адресов:
nmap 192.168.1.0-200
поиск активных хостов:
nmap -sn 192.168.1.0/24
сканирование списка хостов: 
nmap -iL input.txt
исключение цели из скнирования:
nmap 192.168.1.0/24 --exclude 192.168.1.5
один/несколько портов:
nmap -p 80,443 192.168.1.1
все порты:
nmap -p "*" 192.168.1.1
топ самых распространенных портов:
nmap --top-ports 5 192.168.1.1
